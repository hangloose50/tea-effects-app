## Data Collection Methodology

1. **Source Aggregation** ‚Äî Capture peer-reviewed articles, institute white papers, and traditional pharmacopeia references per tea/compound/effect. Store citation metadata (title, author, DOI/URL, publication year) in a structured Zotero collection.
2. **Compound Profiling** ‚Äî Extract quantitative ranges (mg per 240‚ÄØml infusion) from lab analyses. Normalize units, record brewing parameters used in studies, and note variance factors (leaf grade, steep time).
3. **Effect Mapping** ‚Äî Link compounds to effects through mechanistic evidence (e.g., EGCG ‚Üí BDNF upregulation ‚Üí neuroplasticity). Prioritize human trials; animal/in vitro data flagged as ‚Äúpreliminary‚Äù.
4. **Tea Entry Authoring** ‚Äî For each tea, synthesize compound spectrum, brewing best practices, sensory notes, and effect intensities. Cross-check against multiple sources (minimum two independent references).
5. **RAG Content Drafting** ‚Äî Outline for each domain document; populate with layered sections (summary ‚Üí mechanisms ‚Üí protocols). Cite inline and append reference list (APA style with DOI links).
6. **Validation Pass** ‚Äî Run custom validator to ensure schema compliance, numeric bounds, reference integrity (compounds/effects exist), and citation presence.
